# AGENTS.md

このファイルは、このプロジェクトでAIエージェントが作業するときの共通ルールです。

## Codexの役割

Codexはこのプロジェクトの管理運営担当です。
原則として、ユーザーから明確に依頼されない限り、分析プログラム本体を新規実装・大幅改修しません。

主な担当:

- プロジェクト構成の整理
- Git管理、ignore設定、コミット前確認
- データ配置ルールと運用手順の整備
- Notebookやレポートの命名・動線確認
- Claude Codeが開発しやすいように、前提・制約・作業順を明文化する
- 実装結果のレビュー、検証観点、リスク整理

## Claude Codeとの分担

- Claude Code: `CLAUDE.md` を読んで、晴太の分析開発を進める
- Codex: `AGENTS.md` を基準に、プロジェクト全体の管理・運営・品質確認を行う

Claude Codeで開発する場合、Codexは裏方として以下を担当する。

- 変更前後のファイル整理
- Gitに入れてよいもの、入れてはいけないものの確認
- データや出力物が誤ってpushされないかの確認
- 分析方針、検証方法、レポート構成の相談

## Git管理ルール

GitHubに上げるもの:

- 共通README
- 共通Notebook
- `はるた/` の公開可能なNotebook、README、レポート
- `requirements.txt`
- データ配置用のREADMEや `.gitkeep`

GitHubに上げないもの:

- `あつと/`
- `CLAUDE.md`
- `.claude/`
- `Dev/` または `はるた/Dev/`
- 実データ本体
- `outputs/`
- `.DS_Store`
- Notebookの実行出力

NotebookをGit管理する場合は、原則として出力セルと実行番号をクリアしてからコミットする。

## データ運用

実データはローカルに置く。
GitHubにはデータ本体を上げず、配置場所と使い方だけをREADMEで管理する。

標準配置:

- 入力データ: `data/candidates/01_sorted/{dataset_name}/`
- 処理済みデータ: `data/processed/{dataset_name}/`
- 晴太の分析出力: `はるた/outputs/`

## 作業時の確認

作業前:

- `git status --short --branch` で状態を確認する
- 変更対象が `あつと/` や個人用ファイルに入っていないか確認する

作業後:

- `.gitignore` が効いているか確認する
- Notebookに出力セルが残っていないか確認する
- 必要ならコミットしてpushする

## 判断基準

このプロジェクトでは、分析の中身そのものよりも、再現できる運用を優先する。
誰が見ても、どこにデータを置き、どのNotebookを開き、どの出力を確認するかが分かる状態を保つ。
